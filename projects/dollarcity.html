Research Findings

18 friction points. Every one documented.

Contextual inquiry embedded with users during live fraud review — not surveys, not workshops. Watching real work reveal real problems.

Video search overhead
Filters reset every session. Users set the same store/terminal/date from scratch, every time they opened the SaaS.
+8 min/session
Multiple tabs open
3–4 browser tabs to cross-reference video and transaction data. No single source of truth across the workflow.
3–4 tabs
Re-requesting videos
No visual distinction between downloaded and pending. Users re-requested videos already available — wasting bandwidth and time.
+5 min/session
No download feedback
Zero progress indication during downloads. Users couldn't tell if the system was working — or stuck.
0 states shown
User Research

4 roles. 1 platform.
0 shared mental models.

Contextual inquiry surfaced four distinct user types — each with unique workflows, permissions, and definitions of "done."

Reviewer
Watches and classifies video exceptions in real time
3–4 tabs open per case
Auditor
Audits classified exceptions for compliance review
No status visibility across the queue
Senior
Oversees team and escalates critical exceptions
Same UI as junior — no role distinction
Directive
Views reports and fraud metrics only
Report export buried 4 clicks deep
Design Decisions

Three calls that changed the numbers.

01
One Screen Rule
60%+ of task friction eliminated

POS search, video filter, and playback in one scrollable panel. Users went from 3–4 open tabs to a single unified view — same functionality, zero context switching.

BEFORE · 3 tabs
AFTER · 1 panel
02
Download State System
Duplicate requests dropped to near zero in testing

4 explicit states — Not Requested, Downloading, Downloaded, Playing — with visual badges and toast confirmation. Users could trust the queue without polling.

Not requested→
Downloading…→
Downloaded→

03
My Video Request as Default
Largest single drop in time-to-first-action across all user types

The user's own request queue became the landing state. Supervisors accessed a secondary global view via role toggle. One decision, maximum impact on the metric that mattered most.
