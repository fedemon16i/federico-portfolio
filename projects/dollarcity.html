User Research

4 roles. 1 platform.
0 shared mental models.

Contextual inquiry surfaced four distinct user types — each with their own workflow, permission set, and definition of "done."

Reviewer
Watches and classifies video exceptions in real time
3–4 tabs open per case
Auditor
Audits classified exceptions for compliance
No status visibility across the queue
Senior
Oversees team and escalates critical exceptions
Same UI as junior — no role distinction
Directive
Views reports and fraud metrics only
Report export buried 4 clicks deep
Design Decisions

Three calls that changed the numbers.

01
One Screen Rule
60%+ of task friction eliminated

POS search, video filter, and playback in one scrollable panel. Users went from 3–4 open tabs to a single unified view — same functionality, zero context switching.

BEFORE · 3 tabs
AFTER · 1 panel
02
Download State System
Duplicate requests dropped to near zero in testing

4 explicit states — Not Requested, Downloading, Downloaded, Playing — with visual badges and toast confirmation. Users could trust the queue without polling.

03
My Video Request as Default
Largest single drop in time-to-first-action across all user types

The user's own request queue became the landing state. Supervisors got a secondary global view via role toggle. One decision, maximum impact on the metric that mattered most.
